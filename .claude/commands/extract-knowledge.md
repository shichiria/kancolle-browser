# Extract Knowledge

GEMINI/ と LOG/ の会話ログからKanColleドメイン知識を抽出し、docs/KNOWLEDGE/ に追加する。
抽出完了後、処理済みファイルは各フォルダ内の backup/ に退避する。

## 手順

### 1. 既存KNOWLEDGEの把握
- `docs/KNOWLEDGE/README.md` を読み、現在のカバー範囲を把握する

### 2. ログの走査
- `GEMINI/*.md` と `GEMINI/*.txt` を走査（backup/ 内は対象外）
- `LOG/*.md` を走査（backup/ 内は対象外）
- `$ARGUMENTS` が指定された場合はそのファイル/フォルダのみ対象

### 3. ドメイン知識の抽出
以下に該当する内容を抽出する:
- **ゲーム仕様**: APIフィールドの意味、計算式、閾値、艦種判定ルール
- **データ構造**: HP配列のインデックス、装備タイプID、マスターデータのフィールド名
- **ゲームロジック**: 戦闘フェーズ順序、任務条件、遠征条件、OASW判定
- **エッジケース**: 連合艦隊のオフセット、月境界の扱い、四半期年トラップ
- **UI座標/表示**: ゲーム画面の座標系、スプライト配置、ノード分類

以下は**抽出しない**:
- 実装方針・アーキテクチャレビュー（コードの書き方に関する助言）
- バグ修正の手順（修正自体はコードに反映済み）
- CSS/UIリファクタリングの計画
- ツール・環境のトラブルシューティング

### 4. 既存ファイルとの照合
- 抽出した知識が既存KNOWLEDGEファイルに**既に記載済み**でないか確認する
- 重複は追加しない
- 既存の記載が不完全な場合は補完する

### 5. 追加・更新
- 既存ファイルのトピックに該当 → 該当ファイルに追記
- 新規トピック → 新ファイルを作成
- `docs/KNOWLEDGE/README.md` の一覧も更新する

### 6. 処理済みファイルの退避
- 抽出が完了したファイルを各フォルダ内の backup/ に移動する:
  - `GEMINI/*.md` → `GEMINI/backup/`
  - `GEMINI/*.txt` → `GEMINI/backup/`
  - `LOG/*.md` → `LOG/backup/`
- backup/ フォルダがなければ作成する
- 次回実行時に再走査されないようにする


## 注意
- KNOWLEDGEファイルは**事実・仕様のみ**を記載する（実装コードは含めない）
- 出典がGeminiのレビュー回答の場合、ゲーム仕様として正確か判断して記載する
- 不確かな情報は `※未検証` を付記する
